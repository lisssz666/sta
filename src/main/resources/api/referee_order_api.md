# 裁判订单API文档

## 1. 取消裁判订单

### 接口路径
`POST /referee/order/cancel`

### 请求参数
| 参数名 | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| id | Long | 是 | 订单ID |

### 响应示例
#### 成功响应
```json
{
  "code": 200,
  "msg": "订单已取消",
  "data": null
}
```

#### 失败响应
```json
{
  "code": 500,
  "msg": "订单状态不允许取消",
  "data": null
}
```

### 功能说明
1. **订单状态检查**：只有未付款（status=0）或已付款（status=1）的订单可以取消
2. **退款流程**：如果订单已付款（status=1），会执行退款流程
3. **时间段放开**：取消订单后，会从裁判的scheduleLog中移除对应的时间段，使其状态变为可用（status=0）
4. **订单状态更新**：取消成功后，订单状态会更新为已取消（status=2）

## 2. 其他相关接口

### 创建裁判订单
- **接口路径**：`POST /referee/order/create`
- **请求体**：
  ```json
  {
    "refereeId": 1,
    "contactName": "张三",
    "contactPhone": "13800138000",
    "totalAmount": "200",
    "gameId": 1,
    "matchTime": "2026-02-05 14:00-16:00",
    "remark": "备注",
    "matchInfo": "比赛信息",
    "matchLocation": "比赛地点",
    "scheduleLog": "[{\"date\":\"2026-02-05\",\"time\":[\"14:00-16:00\"]}]"
  }
  ```

### 获取订单详情
- **接口路径**：`GET /referee/order/detail`
- **请求参数**：`id`（订单ID）

### 获取订单列表
- **接口路径**：`GET /referee/order/list`
- **请求参数**：
  - `refereeId`（可选，裁判ID）
  - `contactPhone`（可选，联系电话）
  - `status`（可选，订单状态：0未付款 1已付款 2已取消 3已完成）

## 3. 订单状态说明

| 状态值 | 状态名称 | 说明 |
| :--- | :--- | :--- |
| 0 | 未付款 | 订单已创建但尚未支付 |
| 1 | 已付款 | 订单已支付成功 |
| 2 | 已取消 | 订单已取消（包含退款流程） |
| 3 | 已完成 | 订单已完成 |

## 4. 注意事项

1. **取消订单的时间限制**：建议在比赛开始前取消订单，以免影响裁判的安排
2. **退款到账时间**：退款流程执行后，资金通常会在1-3个工作日内退回原支付账户
3. **时间段释放**：取消订单后，对应的时间段会立即释放，其他用户可以再次预约
4. **重复取消**：已取消的订单无法再次取消，会返回错误提示
5. **订单不存在**：如果订单ID不存在，会返回"订单不存在"的错误提示
